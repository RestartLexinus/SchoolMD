
## 1. Структура проекта

Создадим следующие объекты в `ServerScriptService` и `ReplicatedStorage`:

```
ServerScriptService
 ├── WaveManager (Script) – управление волнами и спавном зомби
 ├── CurrencyHandler (Script) – обработка начисления валюты за убийство
 └── ShopHandler (Script) – обработка покупок улучшений

ReplicatedStorage
 ├── Events
 │    └── PurchaseUpgrade (RemoteEvent) – клиент запрашивает покупку
 └── Modules
      └── UpgradeData (ModuleScript) – данные об улучшениях

StarterPlayer
 └── StarterPlayerScripts
      └── ShopUI (LocalScript) – интерфейс магазина
```

## 2. Создание валюты (leaderstats)

Для хранения валюты и уровня волны используем стандартную папку `leaderstats`. Добавьте в `StarterPlayerScripts` локальный скрипт:

```lua
-- StarterPlayerScripts/CreateLeaderstats
local Players = game:GetService("Players")

Players.PlayerAdded:Connect(function(player)
	local leaderstats = Instance.new("Folder")
	leaderstats.Name = "leaderstats"
	leaderstats.Parent = player

	local coins = Instance.new("NumberValue")
	coins.Name = "Coins"
	coins.Value = 0
	coins.Parent = leaderstats

	local wave = Instance.new("NumberValue")
	wave.Name = "Wave"
	wave.Value = 0
	wave.Parent = leaderstats
end)
```

Теперь у каждого игрока будут счётчики монет и текущей волны.

## 3. Модуль с данными улучшений

Создадим `UpgradeData` в `ReplicatedStorage/Modules`. Этот модуль будет содержать информацию о доступных улучшениях и их стоимости. Сервер и клиент смогут его использовать.

```lua
-- ReplicatedStorage/Modules/UpgradeData
local UpgradeData = {}

UpgradeData.Upgrades = {
	-- Название улучшения, свойство, которое меняем, базовая стоимость, прирост стоимости за уровень
	Speed = {
		Name = "Скорость передвижения",
		Property = "WalkSpeed",
		BaseCost = 100,
		CostIncrease = 50,
		MaxLevel = 10,
		Effect = function(level) return 16 + level * 2 end -- начальная скорость 16
	},
	Damage = {
		Name = "Урон",
		Property = "Damage",
		BaseCost = 150,
		CostIncrease = 75,
		MaxLevel = 10,
		Effect = function(level) return 10 + level * 5 end
	},
	Health = {
		Name = "Максимальное здоровье",
		Property = "MaxHealth",
		BaseCost = 80,
		CostIncrease = 40,
		MaxLevel = 10,
		Effect = function(level) return 100 + level * 20 end
	}
}

-- Функция для получения стоимости следующего уровня
function UpgradeData.GetCost(upgradeKey, currentLevel)
	local data = UpgradeData.Upgrades[upgradeKey]
	if not data then return nil end
	return data.BaseCost + (currentLevel * data.CostIncrease)
end

-- Функция для применения улучшения к игроку (серверная)
function UpgradeData.ApplyUpgrade(player, upgradeKey, newLevel)
	local data = UpgradeData.Upgrades[upgradeKey]
	if not data then return false end
	local character = player.Character
	if not character then return false end
	local humanoid = character:FindFirstChild("Humanoid")
	if not humanoid then return false end
	
	if upgradeKey == "Speed" then
		humanoid.WalkSpeed = data.Effect(newLevel)
	elseif upgradeKey == "Damage" then
		-- Здесь предполагается, что у игрока есть оружие с атрибутом Damage
		-- Например, можно сохранить значение в атрибут инструмента или в IntValue внутри игрока
		local tool = character:FindFirstChildOfClass("Tool")
		if tool then
			local damageAttr = tool:GetAttribute("Damage")
			if damageAttr then
				tool:SetAttribute("Damage", data.Effect(newLevel))
			end
		end
	elseif upgradeKey == "Health" then
		humanoid.MaxHealth = data.Effect(newLevel)
		humanoid.Health = humanoid.MaxHealth -- восстанавливаем здоровье
	end
	return true
end

return UpgradeData
```

## 4. Серверный скрипт обработки покупок

Создадим `ShopHandler` в `ServerScriptService`. Он будет слушать запросы от клиента через `RemoteEvent` и применять улучшения, списывая монеты.

```lua
-- ServerScriptService/ShopHandler
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Players = game:GetService("Players")

local UpgradeData = require(ReplicatedStorage:WaitForChild("Modules"):WaitForChild("UpgradeData"))
local purchaseEvent = Instance.new("RemoteEvent")
purchaseEvent.Name = "PurchaseUpgrade"
purchaseEvent.Parent = ReplicatedStorage:WaitForChild("Events")

-- Храним уровни улучшений для каждого игрока
local playerUpgrades = {}

local function getUpgradeLevel(player, upgradeKey)
	if not playerUpgrades[player] then
		playerUpgrades[player] = {}
	end
	return playerUpgrades[player][upgradeKey] or 0
end

local function setUpgradeLevel(player, upgradeKey, level)
	if not playerUpgrades[player] then
		playerUpgrades[player] = {}
	end
	playerUpgrades[player][upgradeKey] = level
end

purchaseEvent.OnServerEvent:Connect(function(player, upgradeKey)
	local leaderstats = player:FindFirstChild("leaderstats")
	if not leaderstats then return end
	local coins = leaderstats:FindFirstChild("Coins")
	if not coins then return end

	local currentLevel = getUpgradeLevel(player, upgradeKey)
	local upgradeData = UpgradeData.Upgrades[upgradeKey]
	if not upgradeData then return end

	if currentLevel >= upgradeData.MaxLevel then
		-- Можно отправить уведомление игроку, что достигнут максимум
		return
	end

	local cost = UpgradeData.GetCost(upgradeKey, currentLevel)
	if coins.Value >= cost then
		coins.Value = coins.Value - cost
		local newLevel = currentLevel + 1
		setUpgradeLevel(player, upgradeKey, newLevel)

		-- Применяем улучшение к персонажу, если он существует
		if player.Character then
			UpgradeData.ApplyUpgrade(player, upgradeKey, newLevel)
		end
	end
end)

-- Также применяем улучшения при появлении персонажа (например, после респавна)
Players.PlayerAdded:Connect(function(player)
	player.CharacterAdded:Connect(function(character)
		-- Применяем все текущие улучшения игрока
		local upgrades = playerUpgrades[player]
		if upgrades then
			for upgradeKey, level in pairs(upgrades) do
				UpgradeData.ApplyUpgrade(player, upgradeKey, level)
			end
		end
	end)
end)
```

## 5. Управление волнами и спавном зомби

Создадим `WaveManager` в `ServerScriptService`. Он будет управлять таймером волн, спавнить зомби в заданных местах и увеличивать сложность.

```lua
-- ServerScriptService/WaveManager
local Players = game:GetService("Players")
local RunService = game:GetService("RunService")

-- Настройки
local SPAWN_POINTS = { -- координаты точек спавна (можно разместить Parts и получить их позиции)
	Vector3.new(10, 0, 20),
	Vector3.new(-15, 0, 25),
	Vector3.new(5, 0, -10)
}
local ZOMBIE_TEMPLATE = game:GetService("ServerStorage"):WaitForChild("Zombie") -- модель зомби
local ZOMBIE_DAMAGE = 20
local BASE_ZOMBIES_COUNT = 3
local EXTRA_ZOMBIES_PER_WAVE = 2
local TIME_BETWEEN_WAVES = 10

local activeZombies = {}
local waveNumber = 0
local waveInProgress = false

local function spawnZombie(position)
	local zombie = ZOMBIE_TEMPLATE:Clone()
	zombie.Parent = workspace
	zombie:SetPrimaryPartCFrame(CFrame.new(position))
	local humanoid = zombie:FindFirstChild("Humanoid")
	if humanoid then
		humanoid.Died:Connect(function()
			-- Убираем из активного списка
			for i, z in ipairs(activeZombies) do
				if z == zombie then
					table.remove(activeZombies, i)
					break
				end
			end
			-- Начисляем валюту убившему игроку (будет через отдельный обработчик)
		end)
	end
	table.insert(activeZombies, zombie)
	return zombie
end

local function startWave()
	if waveInProgress then return end
	waveInProgress = true
	waveNumber = waveNumber + 1

	-- Обновляем значение волны у всех игроков
	for _, player in ipairs(Players:GetPlayers()) do
		local leaderstats = player:FindFirstChild("leaderstats")
		if leaderstats then
			local waveValue = leaderstats:FindFirstChild("Wave")
			if waveValue then
				waveValue.Value = waveNumber
			end
		end
	end

	local zombiesToSpawn = BASE_ZOMBIES_COUNT + (waveNumber - 1) * EXTRA_ZOMBIES_PER_WAVE
	print("Волна " .. waveNumber .. " начинается! Зомби: " .. zombiesToSpawn)

	-- Спавним с небольшой задержкой между появлениями
	for i = 1, zombiesToSpawn do
		local spawnPoint = SPAWN_POINTS[math.random(1, #SPAWN_POINTS)]
		spawnZombie(spawnPoint)
		task.wait(1) -- задержка между спавнами внутри волны
	end

	-- Ждём, пока все зомби будут убиты
	while #activeZombies > 0 do
		task.wait(1)
	end

	waveInProgress = false
	print("Волна " .. waveNumber .. " завершена. Следующая через " .. TIME_BETWEEN_WAVES .. " секунд.")
	task.wait(TIME_BETWEEN_WAVES)
	startWave()
end

-- Запускаем цикл волн через небольшую задержку после старта игры
task.wait(5)
startWave()
```

## 6. Начисление валюты за убийство зомби

Создадим `CurrencyHandler` в `ServerScriptService`. Он будет слушать событие смерти зомби и начислять монеты игроку, нанёсшему последний урон.

```lua
-- ServerScriptService/CurrencyHandler
local Players = game:GetService("Players")
local ReplicatedStorage = game:GetService("ReplicatedStorage")

local REWARD_PER_ZOMBIE = 25

local function onZombieDied(zombie, killer)
	if not killer then return end
	-- killer - это объект Player? Нет, в Humanoid.Died мы не получаем убийцу напрямую.
	-- Используем GetDamageHistory для определения последнего нанёсшего урон игрока.
	local humanoid = zombie:FindFirstChild("Humanoid")
	if not humanoid then return end

	local damageHistory = humanoid:GetDamageHistory()
	local lastPlayer = nil
	for i = #damageHistory, 1, -1 do
		local entry = damageHistory[i]
		if entry and entry.Creator then
			local creator = entry.Creator
			if creator:IsA("Player") then
				lastPlayer = creator
				break
			end
		end
	end

	if lastPlayer then
		local leaderstats = lastPlayer:FindFirstChild("leaderstats")
		if leaderstats then
			local coins = leaderstats:FindFirstChild("Coins")
			if coins then
				coins.Value = coins.Value + REWARD_PER_ZOMBIE
				print(lastPlayer.Name .. " получил " .. REWARD_PER_ZOMBIE .. " монет за зомби")
			end
		end
	end
end

-- Функция для связывания зомби при спавне (можно вызвать из WaveManager)
function RegisterZombie(zombie)
	local humanoid = zombie:FindFirstChild("Humanoid")
	if humanoid then
		humanoid.Died:Connect(function()
			onZombieDied(zombie)
		end)
	end
end

-- Если зомби создаются не только в WaveManager, можно также отслеживать добавление новых моделей
-- через ChildAdded в workspace, но это менее эффективно.
```

Интегрируем это с `WaveManager`, добавив вызов `RegisterZombie` после клонирования:

```lua
-- внутри spawnZombie в WaveManager
local zombie = ZOMBIE_TEMPLATE:Clone()
-- ... позиционирование
-- Вызываем регистрацию для обработки награды
local currencyHandler = require(script.Parent:WaitForChild("CurrencyHandler")) -- или require глобальный
currencyHandler.RegisterZombie(zombie)
```

Чтобы избежать циклических зависимостей, лучше вынести `CurrencyHandler` в отдельный модуль, который можно `require` в `WaveManager`. Для этого создадим `CurrencyHandler` как `ModuleScript` в `ServerScriptService` и вернём функцию `RegisterZombie`. Либо просто скопируем код регистрации в `WaveManager` напрямую – для простоты.

## 7. Создание интерфейса магазина (LocalScript)

Теперь создадим UI для магазина. Например, `ScreenGui` с кнопками улучшений. Разместим его в `StarterGui`, а логику – в `StarterPlayerScripts`.

```lua
-- StarterPlayerScripts/ShopUI (LocalScript)
local Players = game:GetService("Players")
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local player = Players.LocalPlayer

local purchaseEvent = ReplicatedStorage:WaitForChild("Events"):WaitForChild("PurchaseUpgrade")
local UpgradeData = require(ReplicatedStorage:WaitForChild("Modules"):WaitForChild("UpgradeData"))

-- Ждём, пока интерфейс загрузится
local screenGui = player.PlayerGui:WaitForChild("ShopGui") -- предполагаем, что Gui уже создан

-- Создаём кнопки для каждого улучшения
local buttons = {}
for upgradeKey, data in pairs(UpgradeData.Upgrades) do
	local button = Instance.new("TextButton")
	button.Name = upgradeKey
	button.Size = UDim2.new(0, 200, 0, 50)
	button.Position = UDim2.new(0, 10, 0, 10 + (#buttons * 60))
	button.Text = data.Name .. "\nСтоимость: " .. UpgradeData.GetCost(upgradeKey, 0)
	button.Parent = screenGui

	button.MouseButton1Click:Connect(function()
		purchaseEvent:FireServer(upgradeKey)
	end)
	buttons[upgradeKey] = button
end

-- Обновляем текст кнопок при изменении количества монет (опционально)
local leaderstats = player:WaitForChild("leaderstats")
local coins = leaderstats:WaitForChild("Coins")

coins:GetPropertyChangedSignal("Value"):Connect(function()
	for upgradeKey, data in pairs(UpgradeData.Upgrades) do
		-- Для корректного отображения стоимости нужен текущий уровень, но уровень хранится на сервере.
		-- Можно запрашивать с сервера или хранить в реплицируемом значении.
		-- Упростим: просто показываем базовую стоимость, или реализуем репликацию уровней через IntValue.
	end
end)
```

Для отображения текущих уровней улучшений можно создать в `leaderstats` папку `Upgrades` и реплицировать уровни через `NumberValue`. Сервер будет обновлять их при покупке, а клиент – читать и обновлять кнопки.

## 8. Создание модели зомби

Создайте простую модель в `ServerStorage` с именем `Zombie`. Она должна содержать:

- `Humanoid` с параметрами (здоровье 100, скорость 8).
- `Part` как основную часть, с `Anchored = false` и `CanCollide = true`.
- Скрипт атаки (например, прикосновением наносит урон игроку).

Пример скрипта атаки внутри зомби:

```lua
-- Внутри модели Zombie, в Part (например, "Head" или "HumanoidRootPart")
local damage = 20

script.Parent.Touched:Connect(function(hit)
	local humanoid = hit.Parent:FindFirstChild("Humanoid")
	if humanoid and hit.Parent:FindFirstChild("Humanoid") then
		humanoid:TakeDamage(damage)
	end
end)
```

## 9. Тестирование и доработки

1. Убедитесь, что точки спавна зомби находятся на безопасной территории, где игроки могут сражаться.
2. Проверьте, что улучшения корректно применяются к персонажу после покупки и после респавна.
3. Добавьте визуальные эффекты (например, частицы при покупке) и звуки.
4. Оптимизируйте производительность: удаляйте убитых зомби через некоторое время, не оставляйте их в памяти.

## Заключение

Мы создали полностью функциональную систему волн зомби с возможностью прокачки. Основные моменты:

- Вся критическая логика (выдача валюты, улучшения, спавн) находится на сервере для защиты от читов.
- RemoteEvent используется для безопасной передачи запросов на покупку.
- Модули данных упрощают добавление новых улучшений и настройку сложности.
